# Cloudflare Tunnel + Vercel Deploy Guide (iamspiderman.me)

This guide publishes both backends under:
- `https://legal-ai.iamspiderman.me/api/*` (Node backend)
- `https://legal-ai.iamspiderman.me/api/v1/chatbot/*` (Python backend)

Frontend stays unchanged in code. You only change frontend env vars for local vs Vercel.

---

## 1. Prerequisites

1. Domain exists in Cloudflare: `iamspiderman.me`
2. WSL Ubuntu installed and updated
3. Repo available inside WSL filesystem (recommended path: `/home/<user>/justice-ai`)
4. Node.js 18+ and Python 3.10+ in WSL

Commands:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git python3 python3-pip python3-venv
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

---

## 2. Clone / Move Repo into WSL

Use Linux path for better performance and service stability:

```bash
cd ~
git clone <your-repo-url> justice-ai
cd justice-ai
```

If already in Windows path (`/mnt/d/...`), copy to `~/justice-ai` for production.

---

## 3. Install App Dependencies

### Node backend

```bash
cd ~/justice-ai/backend
npm install
cp .env.example.production .env
# Edit .env and set Supabase + JWT + CORS_ORIGIN
```

### Python chatbot backend

```bash
cd ~/justice-ai/chatbot/backend
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install fastapi uvicorn pydantic python-dotenv openai requests
cp .env.example .env
# Edit .env and set GROQ_API_KEY at minimum
deactivate
```

---

## 4. Enable systemd in WSL

Edit `/etc/wsl.conf`:

```ini
[boot]
systemd=true
```

Then from Windows PowerShell:

```powershell
wsl --shutdown
```

Start WSL again and verify:

```bash
systemctl is-system-running
```

---

## 5. Install Backend Services

From repo root:

```bash
cd ~/justice-ai
chmod +x deploy/wsl/install_services.sh
./deploy/wsl/install_services.sh /home/<your-user>/justice-ai <your-user>
```

Check:

```bash
sudo systemctl status justice-node.service
sudo systemctl status justice-chatbot.service
```

Quick local endpoint checks:

```bash
curl http://127.0.0.1:3001/health
curl http://127.0.0.1:8000/health
```

---

## 6. Install cloudflared in WSL

```bash
cd ~
wget -O cloudflared.deb https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared.deb
cloudflared --version
```

---

## 7. Create Cloudflare Tunnel

1. Login:

```bash
cloudflared tunnel login
```

2. Create tunnel:

```bash
cloudflared tunnel create legal-ai
```

This prints a `Tunnel ID` and creates credentials JSON in `~/.cloudflared/`.

3. Configure tunnel:

```bash
mkdir -p ~/.cloudflared
cp ~/justice-ai/deploy/cloudflared/config.yml.example ~/.cloudflared/config.yml
```

Edit `~/.cloudflared/config.yml`:
- Replace `YOUR_TUNNEL_ID`
- Replace `YOUR_WSL_USER`
- Replace credentials JSON filename with the one generated by `cloudflared tunnel create`

4. Map DNS:

```bash
cloudflared tunnel route dns legal-ai legal-ai.iamspiderman.me
```

This creates the required DNS record in Cloudflare automatically.

---

## 8. Run Tunnel as Service

```bash
sudo cloudflared service install
sudo systemctl enable cloudflared
sudo systemctl restart cloudflared
sudo systemctl status cloudflared
```

Test public URLs:

```bash
curl https://legal-ai.iamspiderman.me/health
curl -X POST https://legal-ai.iamspiderman.me/api/v1/chatbot/chat \
  -H "Content-Type: application/json" \
  -d '{"messages":[{"role":"user","content":"hello"}]}'
```

---

## 9. Cloudflare Dashboard Checks

1. DNS:
- Confirm record for `legal-ai` exists and is proxied.

2. SSL/TLS:
- Set mode to `Full` or `Full (strict)`.

3. Zero Trust > Networks > Tunnels:
- `legal-ai` tunnel should be `Healthy`.

---

## 10. Vercel Frontend Env

Set in Vercel project settings:

- `NEXT_PUBLIC_API_URL=https://legal-ai.iamspiderman.me/api`
- `NEXT_PUBLIC_CHATBOT_API_URL=https://legal-ai.iamspiderman.me/api/v1/chatbot`

Then redeploy frontend.

---

## 11. Local Frontend Env

For local development (`frontend/.env.local`):

```env
NEXT_PUBLIC_API_URL=http://localhost:3001/api
NEXT_PUBLIC_CHATBOT_API_URL=http://localhost:8000/api/v1/chatbot
```

---

## 12. Debug Checklist

1. Frontend still hitting old backend:
- Restart Next.js after env changes.

2. Chatbot returns `AI provider failed`:
- Check `chatbot/backend/.env` has valid `GROQ_API_KEY`.
- Restart chatbot service:
  ```bash
  sudo systemctl restart justice-chatbot.service
  ```
- Check logs:
  ```bash
  sudo journalctl -u justice-chatbot.service -n 200 --no-pager
  ```

3. Public URL not reachable:
- Check tunnel status:
  ```bash
  sudo systemctl status cloudflared
  sudo journalctl -u cloudflared -n 200 --no-pager
  ```

4. CORS errors on Vercel frontend:
- Add Vercel URL to `backend/.env` `CORS_ORIGIN` (comma-separated), then restart Node service.
